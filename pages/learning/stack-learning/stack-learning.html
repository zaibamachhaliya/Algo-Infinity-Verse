01

Introduction to Stacks

A Stack is a linear data structure that follows the Last In, First Out (LIFO) principle — the last element added is always the first one removed. Think of a stack of plates: you always add and remove from the top.

Why Learn Stacks?

  • Function call management — the call stack tracks every function invocation
  • Undo / Redo — text editors, Photoshop, browsers use stacks
  • Expression evaluation — compilers parse arithmetic using stacks
  • DFS & backtracking — graph traversal and recursion simulation
  • Interview staple — balanced brackets, next greater element, min-stack

PUSH adds to top

💡

The top of the stack is the only accessible element. You cannot reach elements below it without first removing everything above — this constraint is what makes stacks uniquely powerful for ordered-reversal problems.

02

The LIFO Principle

Last In, First Out means the most recently inserted element is the first to be removed. This is the defining rule of every stack — whether implemented with an array, a linked list, or a language built-in.

LIFO Trace
Push 10  →  Stack: [ 10 ]           top = 0
Push 20  →  Stack: [ 10, 20 ]        top = 1
Push 30  →  Stack: [ 10, 20, 30 ]    top = 2
Pop      →  Returns 30  (LAST IN)    top = 1
Pop      →  Returns 20               top = 0
Pop      →  Returns 10  (FIRST IN, last out)

Real-World Analogy

A browser's Back button is a stack. Every page you visit is pushed onto a history stack. Pressing Back pops the most recent page — you always return to where you just came from, not where you started.

⚡

LIFO is the opposite of a Queue's FIFO (First In, First Out). Choosing between them depends entirely on whether you need the newest or the oldest item next.

03

Core Operations

Every stack exposes exactly five operations. All are O(1) when implemented correctly.

Operation Description Time
push(x) Insert element x onto the top O(1)
pop() Remove and return the top elementO(1)
peek() Return top element without removingO(1)
isEmpty() Check if the stack has no elementsO(1)
size() Return the number of elements O(1)
⚠️

Always call isEmpty() before pop() or peek(). Calling these on an empty stack is a stack underflow error — in C it's undefined behaviour, in Java it throws EmptyStackException.

peek()   →  returns 1  (stack unchanged: [ 5, 3, 1 ])
04

Array Implementation

The simplest way to implement a stack is with a fixed-size array and an integer top that tracks the index of the topmost element. top = -1 means the stack is empty.

C — Array Stack 
#define MAX 100

int stack[MAX];
int top = -1;

// Push
void push(int value) {
    if (top == MAX - 1) {
        printf("Stack Overflow!\n");
        return;
    }
    stack[++top] = value;
}

// Pop
int pop() {
    if (top == -1) {
        printf("Stack Underflow!\n");
        return -1;
    }
    return stack[top--];
}

// Peek
int peek() {
    if (top == -1) {
        printf("Stack is empty.\n");
        return -1;
    }
    return stack[top];
}

// isEmpty
int isEmpty() { return top == -1; }

// Size
int size()    { return top + 1; }
Example Usage
push(10);  push(20);  push(30);
peek()  →  30
pop()   →  30
pop()   →  20
size()  →  1
⚡

Array stacks have a fixed capacity defined at compile time. Use a dynamic array (vector in C++, ArrayList in Java) if you need the stack to grow beyond a known maximum.

05

Linked List Implementation

A linked list stack has no fixed capacity — it grows and shrinks dynamically. The head of the linked list acts as the top of the stack. Both push and pop operate at the head, keeping them O(1).

C — Linked List Stack 
struct Node {
    int data;
    struct Node* next;
};

struct Node* top = NULL;  // empty stack

// Push — insert at head
void push(int value) {
    struct Node* node = (struct Node*)malloc(sizeof(struct Node));
    node->data = value;
    node->next = top;    // new node points to old top
    top = node;          // update top
}

// Pop — remove from head
int pop() {
    if (top == NULL) {
        printf("Stack Underflow!\n");
        return -1;
    }
    int val = top->data;
    struct Node* temp = top;
    top = top->next;
    free(temp);
    return val;
}

// Peek
int peek() {
    return (top != NULL) ? top->data : -1;
}

// isEmpty
int isEmpty() { return top == NULL; }
🔗

Linked list stacks trade the memory overhead of pointers for unlimited dynamic size. The array version uses contiguous memory (better cache performance); the linked version uses heap allocation (no overflow risk).

Practice: Linked Stack

Add a size() function to the linked list stack above that returns the number of elements in O(n) time.

C 
int size() {
    int count = 0;
    struct Node* curr = top;
    while (curr != NULL) {
        count++;
        curr = curr->next;
    }
    return count;
}
// Tip: maintain a separate 'count' variable in push/pop
// to achieve O(1) size() instead.
06

Time & Space Complexity

All core stack operations are O(1) — that is what makes stacks so efficient. The table below compares the two standard implementations.

Operation Array Stack Linked Stack
push O(1)* O(1)
pop O(1) O(1)
peek O(1) O(1)
isEmpty O(1) O(1)
size O(1) O(n)**
Space O(n) O(n) + ptr overhead

* Amortised O(1) for dynamic arrays; worst case O(n) on resize.
** O(1) if you maintain a count variable.

💡

Space complexity for both implementations is O(n) where n is the number of elements. The linked list version uses additional memory per node for the next pointer.

07

Real-World Applications

Stacks appear in almost every layer of modern software. Understanding where they are used helps you recognise when to reach for one in your own code.

Function Call Stack

Every time a function is called, the CPU pushes a stack frame onto the call stack containing the return address, parameters, and local variables. When the function returns, the frame is popped. Recursion depth is literally the depth of this stack — exceed it and you get a stack overflow.

C — Balanced Brackets Check 
// Returns 1 if brackets are balanced, 0 otherwise
int isBalanced(char* expr) {
    char stack[100];
    int top = -1;

    for (int i = 0; expr[i] != '\0'; i++) {
        char ch = expr[i];

        // Push opening brackets
        if (ch == '(' || ch == '[' || ch == '{') {
            stack[++top] = ch;
        }
        // Match closing brackets
        else if (ch == ')' || ch == ']' || ch == '}') {
            if (top == -1) return 0;   // nothing to match

            char open = stack[top--];
            if ((ch == ')' && open != '(') ||
                (ch == ']' && open != '[') ||
                (ch == '}' && open != '{'))
                return 0;
        }
    }
    return top == -1;  // balanced if stack is empty
}

// isBalanced("{[()]}")  → 1 (true)
// isBalanced("{[(])}")  → 0 (false)

Other Common Uses

  • Browser history — Back / Forward navigation
  • Undo / Redo — two stacks (undo stack + redo stack)
  • DFS traversal — explicit stack replaces call-stack recursion
  • Syntax parsing — compilers validate code with stacks
  • Postfix evaluation — calculators use an operand stack
08

Expression Evaluation

One of the most classic stack applications is evaluating postfix (Reverse Polish Notation) expressions. Every operand is pushed; every operator pops two operands, computes, and pushes the result.

Infix vs Postfix

Infix: 3 + 4 * 2 — requires precedence rules and parentheses.
Postfix: 3 4 2 * + — no ambiguity, evaluated left-to-right with a stack.

C — Postfix Evaluator 
int evaluatePostfix(char* expr) {
    int stack[100], top = -1;

    for (int i = 0; expr[i] != '\0'; i++) {
        char ch = expr[i];

        if (ch >= '0' && ch <= '9') {
            stack[++top] = ch - '0';   // push digit
        } else if (ch != ' ') {
            int b = stack[top--];       // pop right operand
            int a = stack[top--];       // pop left operand
            switch (ch) {
                case '+': stack[++top] = a + b; break;
                case '-': stack[++top] = a - b; break;
                case '*': stack[++top] = a * b; break;
                case '/': stack[++top] = a / b; break;
            }
        }
    }
    return stack[top];
}

// evaluatePostfix("3 4 2 * +")
// Step 1: push 3  → [3]
// Step 2: push 4  → [3, 4]
// Step 3: push 2  → [3, 4, 2]
// Step 4: *  → pop 2,4  → push 8   → [3, 8]
// Step 5: +  → pop 8,3  → push 11  → [11]
// Result: 11

09

Stack vs Queue

Stacks and queues are both restricted-access linear structures, but their removal order is opposite. Knowing when to use each is a fundamental DSA skill.

Property Stack Queue
Principle LIFO FIFO
Insert end Top (push) Rear (enqueue)
Remove end Top (pop) Front (dequeue)
Peek Top element Front element
Use case DFS, undo, call stack BFS, scheduling, print queue
Analogy Plate stack Ticket queue
🧩

You can implement a queue using two stacks (and vice versa) — a common interview problem. Push to stack 1; when dequeuing, transfer all elements to stack 2 (reversing order) and pop from stack 2.

Stack Interview Tips

Stack problems are among the most common in technical interviews. They appear in every tier from internships to FAANG senior roles.

Must-Know Problems

  • Daily Temperatures — next greater element pattern (LeetCode #739)

  • Largest Rectangle in Histogram — monotonic stack (LeetCode #84)
  • When you see "next greater / smaller element" → think monotonic stack.
  • When you see "valid brackets / nesting" → always use a stack.
  • Always guard against underflow: check isEmpty() before every pop/peek.
  • Know both array and linked-list implementations — interviewers may ask to code either.
  • The two-stack queue and min stack are nearly guaranteed at top companies.
Classic Interview Problem

Min Stack: Design a stack that supports push, pop, peek, and getMin() — all in O(1) time.

C — Min Stack 
// Use two stacks: one for data, one to track minimums
#define MAX 100
int main_stack[MAX], min_stack[MAX];
int main_top = -1, min_top = -1;

void push(int val) {
    main_stack[++main_top] = val;
    // push to min_stack only if val <= current min
    if (min_top == -1 || val <= min_stack[min_top])
        min_stack[++min_top] = val;
}

int pop() {
    int val = main_stack[main_top--];
    // if popped value was the minimum, pop min_stack too
    if (val == min_stack[min_top])
        min_top--;
    return val;
}

int getMin() {
    return min_stack[min_top];  // O(1) always
}

// push(3) push(5) push(2) push(1)
// getMin() → 1
// pop()    → 1
// getMin() → 2
