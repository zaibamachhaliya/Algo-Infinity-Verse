Why Segment Trees?

If you only need to query sums of a static array, Prefix Sums work in O(1). But what if the array updates? Prefix sums take O(N) to update. A Segment Tree strikes a balance, allowing both queries and updates in O(log N).

Prefix Sums
Query: O(1) Update: O(N)
Segment Tree
Query: O(log N) Update: O(log N)
Brute Force
Query: O(N) Update: O(1)
01

Building the Tree

A Segment Tree is a binary tree where the root represents the whole array [0, N-1]. Each node splits its range in half, delegating to its left and right children. The leaves represent single elements.

Array Representation

We usually store the tree in a 1D array of size 4 * N. For a node at index i:

  • Left child is at 2 * i
  • Right child is at 2 * i + 1
Python (Sum Tree)
def build(node, start, end, arr, tree):
    if start == end:
        tree[node] = arr[start]
        return
    mid = (start + end) // 2
    build(2 * node, start, mid, arr, tree)
    build(2 * node + 1, mid + 1, end, arr, tree)
    tree[node] = tree[2 * node] + tree[2 * node + 1] # Or min/max
C++ (Sum Tree)
void build(int node, int start, int end, vector<int>& arr, vector<int>& tree) {
    if (start == end) {
        tree[node] = arr[start];
        return;
    }
    int mid = (start + end) / 2;
    build(2 * node, start, mid, arr, tree);
    build(2 * node + 1, mid + 1, end, arr, tree);
    tree[node] = tree[2 * node] + tree[2 * node + 1];
}
02

Range Queries

To answer a query for range [L, R], we traverse the tree. There are three overlap conditions between the node's range [start, end] and the query range [L, R].

🎯

1. Partial Overlap: Recurse down to both children.

2. Total Overlap: Node range is completely inside [L, R]. Return node value immediately.

3. No Overlap: Node range is completely outside [L, R]. Return a null value (e.g., 0 for sum, infinity for min).

03

Lazy Propagation (Range Updates)

If we want to update a whole range (e.g., add 5 to indices 2 through 8), a standard update would take O(N). Lazy Propagation optimizes this back to O(log N).

The Core Idea

When a node's range is completely covered by the update range, we update the node and stop recursing. We record the update in a lazy[] array for its children, promising to pass the update down ("propagate") only when those children are visited in future queries or updates.

Python
def update_range(node, start, end, l, r, val, tree, lazy):
    # 1. Resolve pending lazy updates
    if lazy[node] != 0:
        tree[node] += (end - start + 1) * lazy[node] # Range sum logic
        if start != end:
            lazy[2 * node] += lazy[node]
            lazy[2 * node + 1] += lazy[node]
        lazy[node] = 0

    # 2. No overlap
    if start > end or start > r or end < l:
        return

    # 3. Total overlap
    if l <= start and end <= r:
        tree[node] += (end - start + 1) * val
        if start != end:
            lazy[2 * node] += val
            lazy[2 * node + 1] += val
        return

    # 4. Partial overlap
    mid = (start + end) // 2
    update_range(2 * node, start, mid, l, r, val, tree, lazy)
    update_range(2 * node + 1, mid + 1, end, l, r, val, tree, lazy)
    tree[node] = tree[2 * node] + tree[2 * node + 1]
C++
void updateRange(int node, int start, int end, int l, int r, int val, vector<int>& tree, vector<int>& lazy) {
    if (lazy[node] != 0) {
        tree[node] += (end - start + 1) * lazy[node];
        if (start != end) {
            lazy[2 * node] += lazy[node];
            lazy[2 * node + 1] += lazy[node];
        }
        lazy[node] = 0;
    }
    if (start > end || start > r || end < l) return;

    if (l <= start && end <= r) {
        tree[node] += (end - start + 1) * val;
        if (start != end) {
            lazy[2 * node] += val;
            lazy[2 * node + 1] += val;
        }
        return;
    }
    int mid = (start + end) / 2;
    updateRange(2 * node, start, mid, l, r, val, tree, lazy);
    updateRange(2 * node + 1, mid + 1, end, l, r, val, tree, lazy);
    tree[node] = tree[2 * node] + tree[2 * node + 1];
}
