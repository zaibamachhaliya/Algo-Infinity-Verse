Introduction to Backtracking

Backtracking is an algorithmic-technique for solving problems recursively by trying to build a solution incrementally, one piece at a time, removing those solutions that fail to satisfy the constraints of the problem at any point of time.

Real-World Analogy

Imagine navigating a maze. You pick a path and follow it. If you hit a dead end, you retrace your steps (backtrack) to the previous junction and try a different path. You keep doing this until you find the exit or explore all paths.

Backtracking is an optimized version of brute-force search. It explores all possible solutions but uses bounding/pruning conditions to stop exploring paths that can never lead to a valid solution.

02

State Space Tree Visualization

A State Space Tree represents all possible states (partial solutions) of a problem. The root represents the initial state, intermediate nodes are partial solutions, and leaf nodes are either full solutions or invalid states.

Subset Generation

Generating all subsets of a given set (Power Set) is a classic backtracking problem. At each element, we have two choices: include it in our current subset, or exclude it.

Python 
def subsets(nums):
    res = []
    
    def backtrack(start, path):
        res.append(list(path))
        
        for i in range(start, len(nums)):
            path.append(nums[i])        # Choose
            backtrack(i + 1, path)      # Explore
            path.pop()                  # Unchoose (Backtrack)
            
    backtrack(0, [])
    return res
Java 
public List<List<Integer>> subsets(int[] nums) {
    List<List<Integer>> res = new ArrayList<>();
    backtrack(res, new ArrayList<>(), nums, 0);
    return res;
}

private void backtrack(List<List<Integer>> res, List<Integer> path, int[] nums, int start) {
    res.add(new ArrayList<>(path));
    
    for (int i = start; i < nums.length; i++) {
        path.add(nums[i]);           // Choose
        backtrack(res, path, nums, i + 1); // Explore
        path.remove(path.size() - 1);// Unchoose (Backtrack)
    }
}
C++ 
vector<vector<int>> subsets(vector<int>& nums) {
    vector<vector<int>> res;
    vector<int> path;
    backtrack(res, path, nums, 0);
    return res;
}

void backtrack(vector<vector<int>>& res, vector<int>& path, vector<int>& nums, int start) {
    res.push_back(path);
    
    for (int i = start; i < nums.size(); i++) {
        path.push_back(nums[i]);    // Choose
        backtrack(res, path, nums, i + 1); // Explore
        path.pop_back();            // Unchoose (Backtrack)
    }
}

Permutations

Generating permutations requires exploring all possible orderings. To avoid using the same element multiple times in a single permutation, we keep track of which elements have been used, or simply swap elements in place.

Python 
def permute(nums):
    res = []
    
    def backtrack(path, used):
        if len(path) == len(nums):
            res.append(list(path))
            return
            
        for i in range(len(nums)):
            if not used[i]:
                used[i] = True
                path.append(nums[i])
                backtrack(path, used)
                path.pop()
                used[i] = False
                
    backtrack([], [False] * len(nums))
    return res
Java 
public List<List<Integer>> permute(int[] nums) {
    List<List<Integer>> res = new ArrayList<>();
    backtrack(res, new ArrayList<>(), nums, new boolean[nums.length]);
    return res;
}

private void backtrack(List<List<Integer>> res, List<Integer> path, int[] nums, boolean[] used) {
    if (path.size() == nums.length) {
        res.add(new ArrayList<>(path));
        return;
    }
    for (int i = 0; i < nums.length; i++) {
        if (used[i]) continue;
        used[i] = true;
        path.add(nums[i]);
        backtrack(res, path, nums, used);
        path.remove(path.size() - 1);
        used[i] = false;
    }
}
C++ 
vector<vector<int>> permute(vector<int>& nums) {
    vector<vector<int>> res;
    vector<int> path;
    vector<bool> used(nums.size(), false);
    backtrack(res, path, nums, used);
    return res;
}

void backtrack(vector<vector<int>>& res, vector<int>& path, vector<int>& nums, vector<bool>& used) {
    if (path.size() == nums.size()) {
        res.push_back(path);
        return;
    }
    for (int i = 0; i < nums.size(); i++) {
        if (used[i]) continue;
        used[i] = true;
        path.push_back(nums[i]);
        backtrack(res, path, nums, used);
        path.pop_back();
        used[i] = false;
    }
}

The N-Queens Problem

The N-Queens problem asks us to place N chess queens on an N×N chessboard so that no two queens threaten each other. This means no two queens can share the same row, column, or diagonal.

Constraint Checking (Pruning)

We place queens row by row. Before placing a queen in a specific column of the current row, we check if it's safe. If it's not safe, we prune that branch of the state space tree immediately, avoiding unnecessary recursive calls.

Python 
def solveNQueens(n):
    res = []
    board = [['.'] * n for _ in range(n)]
    
    col = set()
    posDiag = set()  # (r + c)
    negDiag = set()  # (r - c)
    
    def backtrack(r):
        if r == n:
            copy = ["".join(row) for row in board]
            res.append(copy)
            return
            
        for c in range(n):
            if c in col or (r + c) in posDiag or (r - c) in negDiag:
                continue  # PRUNING
                
            col.add(c)
            posDiag.add(r + c)
            negDiag.add(r - c)
            board[r][c] = 'Q'
            
            backtrack(r + 1)
            
            col.remove(c)
            posDiag.remove(r + c)
            negDiag.remove(r - c)
            board[r][c] = '.'
            
    backtrack(0)
    return res
Java 
public List<List<String>> solveNQueens(int n) {
    List<List<String>> res = new ArrayList<>();
    char[][] board = new char[n][n];
    for (int i = 0; i < n; i++) Arrays.fill(board[i], '.');
    
    backtrack(res, board, 0, n);
    return res;
}

private void backtrack(List<List<String>> res, char[][] board, int r, int n) {
    if (r == n) {
        List<String> copy = new ArrayList<>();
        for (char[] row : board) copy.add(new String(row));
        res.add(copy);
        return;
    }
    
    for (int c = 0; c < n; c++) {
        if (isSafe(board, r, c, n)) {
            board[r][c] = 'Q';
            backtrack(res, board, r + 1, n);
            board[r][c] = '.';
        }
    }
}
C++ 
vector<vector<string>> solveNQueens(int n) {
    vector<vector<string>> res;
    vector<string> board(n, string(n, '.'));
    vector<int> col(n, 0), posDiag(2*n-1, 0), negDiag(2*n-1, 0);
    
    auto backtrack = [&](auto& self, int r) -> void {
        if (r == n) {
            res.push_back(board);
            return;
        }
        for (int c = 0; c < n; c++) {
            if (col[c] || posDiag[r + c] || negDiag[r - c + n - 1]) continue;
            
            col[c] = posDiag[r + c] = negDiag[r - c + n - 1] = 1;
            board[r][c] = 'Q';
            self(self, r + 1);
            board[r][c] = '.';
            col[c] = posDiag[r + c] = negDiag[r - c + n - 1] = 0;
        }
    };
    backtrack(backtrack, 0);
    return res;
}

Sudoku Solver

Sudoku is another popular application of backtracking. We try filling an empty cell with a digit from 1 to 9. If the digit is valid, we recursively try to fill the next empty cell. If we hit a cell where no digit works, we backtrack and change the previously placed digit.

Validation Rules

  • The digit must not exist in the same row.
  • The digit must not exist in the same column.
  • The digit must not exist in the corresponding 3x3 sub-box.
07

Pruning Strategies

Pruning is what makes backtracking faster than pure brute-force search. It involves detecting early that a partial solution cannot possibly be extended to a valid final solution, and stopping the search along that path.

Techniques for Avoiding Redundancy

1. Forward Checking: Maintain domains of legal values for unassigned variables and prune when a domain becomes empty.

2. Symmetry Breaking: If a problem has symmetrical solutions, restrict the search space to only find one of them.

3. Memoization / DP: If state spaces overlap, cache the results of subproblems to avoid redundant recursion paths.

08

Complexity Analysis

Backtracking algorithms are fundamentally exponential in time, but pruning can drastically reduce the number of visited states in practice.

Time & Space Complexity

  • Subsets: Time O(2^N), Space O(N) for recursion stack.
  • Permutations: Time O(N * N!), Space O(N).
  • N-Queens: Time O(N!), Space O(N). (Strictly less than N^N due to pruning).
  • Sudoku: Time O(9^(EmptyCells)), Space O(81) = O(1).
