Getting Started with Greedy Algorithms

In Greedy Algorithm, a locally optimal choice is made at each step with the hope of finding a globally optimum solution.

Unlike Dynamic Programming, which explores multiple possibilities, Greedy Algorithms make decisions immediately and never reconsider them. Hence, they're considered more efficient and easy to implement for many optimization problems.

Key Idea:

Make the best choice available right now and move forward

Real-Life Example

Imagine you are selecting activities to fit into a day. Choosing the activity that finishes earliest leaves maximum room for future activities. This is a classic greedy strategy.

02

Characteristics of Greedy Algorithms

Greedy Algorithms make decisions step-by-step, choosing the best immediate option without reconsidering previous choices.

Main Characteristics

  • Make locally optimal choices at each step.
  • Do not revisit previous decisions.
  • Usually faster and simpler than Dynamic Programming.
  • May not always produce the optimal solution.

Two Important Properties

Greedy Choice Property:

A globally optimal solution can be reached by making locally optimal choices.

Optimal Substructure:

The optimal solution can be built from optimal solutions of smaller subproblems.

Important Note

Not every optimization problem can be solved using a Greedy approach. Some problems require Dynamic Programming because local optimal choices do not always lead to a globally optimal solution.

Assign Cookies

Imagine you're an amazing parent to your children and you want to give them some cookies. But each child should get at most one cookie. Given two arrays, children and cookie, the ith value in the children array describes the minimum size of cookie that the ith child can be assigned. The jth value in the cookie array represents the size of the jth cookie. If cookie[j] >= children[i], the jth cookie can be assigned to the ith child. Maximize the number of your content children and output the maximum number.

Example

g = [1, 2, 3]

s = [1, 1]

You have 3 children and 2 cookies. The greed factors of 3 children are 1, 2, 3. And even though you have 2 cookies, since their size is both 1, you could only make the child whose greed factor is 1 content. You need to output 1.

Algorithm

  1. Sort both children and cookie array in ascending order.
  2. Start with the first child and first cookie. Using the two-pointer approach, traverse the array.
  3. Check if current cookie can satisfy the child's greed, if so, assign the cookie to the child and move to the next child and next cookie.
  4. If not, move to the next cookie.
  5. Count the number of children who've been satisfied. Return the count.

Fractional Knapsack

Fractional Knapsack problem allows us to take fractions of items. Each item has a value and a weight. The goal is to maximize the total value placed inside a knapsack with limited capacity.

Unlike the 0/1 Knapsack problem, we are allowed to take only a part of an item if needed.

Example

value = [60, 100, 120]

weight = [10, 20, 30]

capacity = 50

- Take item 0 (weight = 10, value = 60) - Take item 1 (weight = 20, value = 100) - Take 2⁄3 of item 2 (weight = 20, value = 80) Total value = 60 + 100 + 80 = 240

Algorithm

  1. Sort the items in decreasing order based on their value-to-weight ratio.
  2. Iterate through the items and pick those whose weight is less than or equal to the current capacity of the knapsack.
  3. If an item's weight exceeds the remaining capacity, break the item into smaller units, calculate its value according to the remaining capacity, and add this value to the total.

                    def __init__(self, value, weight):

                    def knapsack(W, arr, n):
                        arr.sort(key=lambda x: (x.value / x.weight), reverse=True)

                        current = 0
                        res = 0.0

                        for i in range(n):
                            if current + arr[i].weight <= W:
                                current = current + arr[i].weight
                                res = res + arr[i].value
                                
                            else:
                                remaining = W - current
                                res = res + (arr[i].value / arr[i].weight) * remaining
                                break

                        return res

Job Sequencing

You are given a set of N jobs where each job comes with a deadline and profit. The profit can only be earned upon completing the job within its deadline. Find the number of jobs done and the maximum profit that can be obtained. Each job takes a single unit of time and only one job can be performed at a time.

Example

N = 4

Jobs = {(1, 4, 20), (2, 1, 10), (3, 1, 40), (4, 1, 30)}

- The 3rd job with a deadline of 1 is performed during the first unit of time. - The 1st job is performed during the second unit of time as its deadline is 4. Profit = 40 + 20 = 60. So, the result is 2 jobs with a total profit of 60.

Algorithm

  1. Sort the jobs in descending order.
  2. Create an array of size x if the maximum deadline is x.
  3. Initialize each array index to -1 to represent that no jobs have been performed yet.
  4. Check if each job can be performed on its last day.
  5. If so, mark that index with the job ID and add the profit to the total.
  6. If not, loop through the previous indexes to find an empty slot and schedule the job there.

                    def __init__(self, id, dead, profit):
                        self.id = id
                        self.dead = dead
                        self.profit = profit
                        
                    def job(arr, n):
                        arr.sort(key=lambda x: x.profit, reverse=True)

                        max_deadline = arr[0].dead

                        for i in range(1, n):
                            max_deadline = max(max_deadline, arr[i].dead)
                            
                        slot = [-1]*(max_deadline + 1)

                        count = 0
                        job_profit = 0

                        for i in range(n):
                            for j in range(arr[i].dead, 0, -1):
                                if slot[j] == -1:
                                    slot[j] = i
                                    count = count + 1

                                    job_profit = job_profit + arr[i].profit

                                    break

                        return count, job_profit

                    import java.util.*;
                        
                    class Job {
                        int id;
                        int dead;
                        int profit;
                            
                        Job(int id, int dead, int profit) {
                            this.id = id;
                            this.dead = dead;
                            this.profit = profit;
                        }
                    }
                        
                    public class JobSequencing {
                        static int[] job(Job[] arr, int n) {
                            Arrays.sort(arr, (a, b) -> b.profit - a.profit);
                                
                            int max_deadline = arr[0].dead;
                                
                            for (int i = 1; i < n; i++) {
                                max_deadline = Math.max(max_deadline, arr[i].dead);
                            }
                                
                            int[] slot = new int[max_deadline + 1];
                            Arrays.fill(slot, -1);
                                
                            int count = 0;
                            int job_profit = 0;
                                
                            for (int i = 0; i < n; i++) {
                                for (int j = arr[i].dead; j > 0; j--) {
                                    if (slot[j] == -1) {
                                        slot[j] = i;
                                        count++;
                                        job_profit = job_profit + arr[i].profit;
                                        break;
                                    }
                                }
                            }
                                
                            return new int[] {
                                count, job_profit
                            };
                        }
                    }

Time Complexity

O(N log N) + O(N * M) - O(N log N) for sorting the jobs in decreasing order of profit. O(N * M) since we are iterating through all N jobs and for every job, we are checking from the last deadline, say M deadlines in the worst case.

Space Complexity

O(M) - For an array that keeps track of which day each job is performed if M is the maximum deadline available.

Activity Selection

Given a set of activities, each with a start time and a finish time, represented by the arrays start[] and finish[], respectively. A single person can perform only one activity at a time, meaning no two activities can overlap. Determine the maximum number of activities that a person can complete in a day.

Example

start[] = [1, 3, 0, 5, 8, 5]

finish[] = [2, 4, 6, 7, 9, 9]

A person can perform at most four activities. The maximum set of activities that can be executed is {0, 1, 3, 4}

Algorithm

  1. Sort the activities by finish time.
  2. Select the first activity.
  3. For each remaining activity, if its start time is >= previous finish time, select it
  4. Count the selected number of activities.

                    import java.util.*;
                        
                    class Activity {
                        static int activity(int[] start, int[] finish) {
                            int n = start.length;
                                
                            int[][] activities = new int[n][2];
                                
                            for (int i = 0; i < n; i++) {
                                activities[i][0] = start[i];
                                activities[i][1] = finish[i];
                            }
                                
                            Arrays.sort(activities, (a, b) -> a[1] - b[1]);
                                
                            int count = 1;
                            int last_finish = activities[0][1];
                                
                            for (int i = 1; i < n; i++) {
                                if (activities[i][0] >= last_finish) {
                                    count++;
                                    last_finish = activities[i][1];
                                }
                            }
                                
                            return count;
                        }
                    }
C++ Code 

                    #include <iostream>
                    #include <vector>
                    #include <algorithm>
                            
                    using namespace std;
                        
                    int activity(vector<int>& start, vector<int>& finish) {
                        vector<pair<int, int>> activities;

                        int x = start.size();
                                
                        for (int i = 0; i < x; i++) {
                            activities.push_back({
                                finish[i], start[i]
                            });
                        }
                            
                        sort(activities.begin(), activities.end());
                            
                        int count = 1;
                        int last_finish = activities[0].first;

                        int y = activities.size();
                            
                        for (int i = 1; i < y; i++) {
                            if (activities[i].second >= last_finish) {
                                count++;
                                last_finish = activities[i].first;
                            }
                        }
                            
                        return count;
                    }

Time Complexity

O(N log N) - The activities must be sorted by their finish times before selection.

Space Complexity

O(N) - Only a few variables are used apart from the input data.

Exercise 1: Lemonade Change

At a lemonade stand, each lemonade costs $5. Customers are standing in a queue to buy from you and order one at a time (in the order specified by bills). Each customer will only buy one lemonade and pay with either a $5, $10, or $20 bill. You must provide the correct change to each customer so that the net transaction is that the customer pays $5. Note that you do not have any change in hand at first. Given an integer array bills where bills[i] is the bill the ith customer pays, return true if you can provide every customer with the correct change, or false otherwise.

Input: bills = [5, 5, 5, 10, 20]

Output: true

Answer:


                        bool lemonadeChange(vector<int>& bills) {
                            int five = 0;
                            int ten = 0;
                            
                            for (int i : bills) {
                                if (i == 5) {
                                    five++;
                                }
                                
                                else if (i == 10) {
                                    if (five > 0) {
                                        ten++;
                                        five--;
                                    }
                                    
                                    else {
                                        return false;
                                    }
                                }
                                
                                else {
                                    if (five > 0 && ten > 0) {
                                        five--;
                                        ten--;
                                    }
                                    
                                    else if (five >= 3) {
                                        five = five - 3;
                                    }
                                    
                                    else {
                                        return false;
                                    }
                                }
                            }
                            
                            return true;
                        }

Exercise 2: Candy

There are n children standing in a line. Each child is assigned a rating value given in the integer array ratings. You're giving candies to these children subjected to the following requirements: 1. Each child must have at least one candy. 2. Children with a higher rating get more candies than their neighbors. Return the minimum number of candies you need to have to distribute the candies to the children.

Input: ratings = [1, 0, 2]

Output: 5

Answer:


                        int candy(vector<int>& ratings) {
                            int n = ratings.size();
                            vector<int> candy(n, 1);
                            
                            for (int i = 1; i < n; i++) {
                                if (ratings[i] > ratings[i-1]) {
                                    candy[i] = candy[i-1] + 1;
                                }
                            }
                            
                            for (int i = n-2; i >= 0; i--) {
                                if (ratings[i] > ratings[i + 1]) {
                                    candy[i] = max(candy[i], candy[i + 1] + 1);
                                }
                            }
                            
                            int total = 0;
                            
                            for (int j : candy) {
                                total = total + j;
                            }
                            
                            return total;
                        }
