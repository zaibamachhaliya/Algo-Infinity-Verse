Introduction to Heaps

A heap is a complete binary tree that satisfies the heap property: in a max-heap, every parent is ≥ its children; in a min-heap, every parent is ≤ its children. Heaps are stored as arrays — no pointers needed.

Why Heaps?

  • O(log n) insert and extract — faster than sorting for streaming data
  • O(1) peek at min/max — always at index 0
  • Top-K in O(n log k) — process n elements with a heap of size k
  • Foundation of Dijkstra's, Prim's, Huffman coding

The array representation of a heap uses index math: for a node at index i:

JavaScript
// Index relationships in heap array
parent     = Math.floor((i - 1) / 2)
leftChild  = 2 * i + 1
rightChild = 2 * i + 2
Example: Min-Heap Array
[1, 3, 6, 5, 9, 8]  →  Tree root is 1 (minimum)
Practice: Identify Heap Type

Given array [4, 8, 6, 10, 12, 9], is this a min-heap or max-heap? What is the parent of index 3?

Answer
// Min-heap: every parent ≤ children
// 4 < 8 ✓, 4 < 6 ✓, 8 < 10 ✓, 8 < 12 ✓, 6 < 9 ✓
parent(3) = Math.floor((3 - 1) / 2) = 1  // value = 8

Min Heap vs Max Heap

The only difference is the ordering property. The same array can be a min-heap or max-heap depending on the comparator used.

PropertyMin HeapMax Heap
RootMinimum elementMaximum element
Parent ruleparent ≤ childrenparent ≥ children
extractMin/MaxO(log n)O(log n)
Use caseDijkstra's, Top-K largest, schedulingHeap sort, Top-K smallest
JS built-inNot natively — simulate with negationNot natively
💡

JavaScript trick: JS has no built-in heap. To simulate a max-heap using min-heap logic, negate all values on insert and negate back on extract.

JavaScript
// Simulate max-heap by negating values
minHeap.push(-value);           // insert negative
const max = -minHeap.pop();     // negate back = max
Practice: When to Use Which

You need the K largest elements from a stream of numbers. Should you use a min-heap or max-heap of size K? Why?

Answer
// Use a MIN-heap of size K
// Keep the K largest seen so far
// If new element > heap root, pop root and push new
// The root is always the K-th largest = threshold

Heap Operations

All heap operations maintain the heap property through sift-up (after insert) and sift-down (after extract).

Operation Complexity

  • insert (push): O(log n) — sift up from last position
  • extractMin/Max (pop): O(log n) — replace root with last, sift down
  • peek: O(1) — root is always min/max
  • heapify (build from array): O(n) — not O(n log n)!
JavaScript
class MinHeap {
  constructor() { this.heap = []; }
  peek()  { return this.heap[0]; }
  size()  { return this.heap.length; }

  push(val) {
    this.heap.push(val);
    this._siftUp(this.heap.length - 1);
  }

  pop() {
    const min = this.heap[0];
    const last = this.heap.pop();
    if (this.heap.length > 0) {
      this.heap[0] = last;
      this._siftDown(0);
    }
    return min;
  }

  _siftUp(i) {
    while (i > 0) {
      const p = Math.floor((i - 1) / 2);
      if (this.heap[p] > this.heap[i]) {
        [this.heap[p], this.heap[i]] = [this.heap[i], this.heap[p]];
        i = p;
      } else break;
    }
  }

  _siftDown(i) {
    const n = this.heap.length;
    while (true) {
      let s = i;
      const l = 2*i+1, r = 2*i+2;
      if (l < n && this.heap[l] < this.heap[s]) s = l;
      if (r < n && this.heap[r] < this.heap[s]) s = r;
      if (s === i) break;
      [this.heap[s], this.heap[i]] = [this.heap[i], this.heap[s]];
      i = s;
    }
  }
}

Priority Queue

A priority queue is an abstract data type where each element has a priority. Elements are dequeued in priority order, not insertion order. Heaps are the most efficient implementation.

Priority Queue vs Regular Queue

  • Queue: FIFO — first in, first out
  • Priority Queue: highest priority element out first
  • Implementation: Min-heap for lowest-priority-first, max-heap for highest-priority-first
JavaScript
// Task scheduler — lower number = higher priority
const pq = new MinHeap();
pq.push([1, 'Critical bug fix']);
pq.push([3, 'Feature request']);
pq.push([2, 'Code review']);
while (pq.size()) {
  const [pri, task] = pq.pop();
  console.log(`Processing: ${task}`);
}
// Critical bug fix → Code review → Feature request
Practice: Dijkstra Setup

In Dijkstra's algorithm, what is stored in the priority queue and what determines priority?

Answer

// Priority = accumulated distance from source
pq.push([0, startNode]);
const [dist, node] = pq.pop();
// Always expand node with smallest distance first

Top-K Frequent Elements

One of the most common heap interview problems. Given an array, find the K most frequent elements. Sorting is O(n log n) — heaps let us do O(n log k).

JavaScript
function topKFrequent(nums, k) {
  const freq = new Map();
  for (const n of nums)
    freq.set(n, (freq.get(n) || 0) + 1);

  const minHeap = new MinHeap(); // [frequency, value]
  for (const [val, count] of freq) {
    minHeap.push([count, val]);
    if (minHeap.size() > k) minHeap.pop(); // evict least frequent
  }
  return minHeap.heap.map(([, val]) => val);
}

// Time: O(n log k)  Space: O(n + k)
Example
topKFrequent([1,1,1,2,2,3], 2)  →  [1, 2]
Practice: K Closest Points

Find K closest points to the origin from an array of points. Write the approach using a max-heap.

JavaScript
function kClosest(points, k) {
  const dist = ([x, y]) => x*x + y*y;
  // Max-heap of size K — evict farthest
  const maxHeap = [];
  for (const p of points) {
    maxHeap.push([dist(p), p]);
    maxHeap.sort((a, b) => b[0] - a[0]);
    if (maxHeap.length > k) maxHeap.shift();
  }
  return maxHeap.map(([, p]) => p);
}

Merge K Sorted Lists

Given K sorted arrays (or linked lists), merge them into one sorted output. Naive approach: merge one by one in O(nK). Heap approach: O(n log K).

Algorithm

  • Push the first element of each list into a min-heap with its list index and position
  • Extract the min — add to result
  • Push the next element from the same list into the heap
  • Repeat until heap is empty
JavaScript
function mergeKSorted(lists) {
  const result = [];
  const heap = new MinHeap(); // [value, listIdx, elemIdx]
  for (let i = 0; i < lists.length; i++)
    if (lists[i].length > 0)
      heap.push([lists[i][0], i, 0]);

  while (heap.size()) {
    const [val, li, ei] = heap.pop();
    result.push(val);
    if (ei + 1 < lists[li].length)
      heap.push([lists[li][ei + 1], li, ei + 1]);
  }
  return result;
}
// Time: O(n log K)  Space: O(K)
Example
mergeKSorted([[1,4,7],[2,5,8],[3,6,9]])  →  [1,2,3,4,5,6,7,8,9]
Practice: Why O(n log K)?

Explain why merging K sorted arrays with a heap is O(n log K) where n is the total number of elements.

Answer
// n = total elements, K = number of lists
// Heap size ≤ K at all times (one entry per list)
// Each element: 1 push + 1 pop = 2 × O(log K)
// Total n elements → O(n log K) ✅

Median of Data Stream

Find the median of a growing stream of numbers in O(log n) per insertion. Maintain two heaps: a max-heap for the lower half and a min-heap for the upper half.

Two-Heap Strategy

  • maxHeap (left half) — stores the smaller half of numbers
  • minHeap (right half) — stores the larger half of numbers
  • Invariant: |maxHeap.size - minHeap.size| ≤ 1
  • Median: If equal sizes → average of both tops. Else → top of larger heap
JavaScript
class MedianFinder {
  constructor() {
    this.lo = new MaxHeap(); // lower half
    this.hi = new MinHeap(); // upper half
  }

  addNum(num) {
    this.lo.push(num);
    this.hi.push(this.lo.pop()); // push lo's max to hi
    if (this.lo.size() < this.hi.size())
      this.lo.push(this.hi.pop()); // rebalance
  }

  findMedian() {
    if (this.lo.size() > this.hi.size())
      return this.lo.peek();
    return (this.lo.peek() + this.hi.peek()) / 2;
  }
}
// addNum: O(log n)  findMedian: O(1)
Practice: Trace

Add [5, 15, 1, 3] one by one using MedianFinder. What is the median after each insertion?

Trace
Add 5:  lo=[5],    hi=[]      median = 5
Add 15: lo=[5],    hi=[15]   median = (5+15)/2 = 10
Add 1:  lo=[5,1],  hi=[15]   median = 5
Add 3:  lo=[3,1],  hi=[5,15] median = (3+5)/2 = 4

Heap vs Sorting — When to Use Which

Both heaps and sorting can solve many of the same problems. Knowing when to choose each is what interviewers test.

ScenarioSortingHeapWinner
Top-K from static arrayO(n log n)O(n log k)🏆 Heap (if k ≪ n)
Top-K from data streamRe-sort each timeO(log k) per insert🏆 Heap
Fully sorted output neededO(n log n)O(n log n)🏆 Sort (simpler)
K-th largest elementO(n log n)O(n log k)🏆 Heap
Merge K sorted listsO(n log n)O(n log k)🏆 Heap
Median of streamNot applicableO(log n) per insert🏆 Heap
🎯

Interview rule of thumb: If the problem involves K or streaming data, reach for a heap first. If you need the full sorted output, sort is simpler.

Practice: Decision

Given 1 billion numbers on disk and you need the top 100 — sort or heap? Why?

Answer
// Min-heap of size 100 — O(n log 100) ≈ O(n)
// Memory: O(100) = O(1) — never load all 1B into RAM
// Sorting 1B elements requires O(n) memory → impractical
// Heap streams data → processes without loading all at once ✅
