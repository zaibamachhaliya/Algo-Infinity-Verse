GCD & LCM

Greatest Common Divisor · Least Common Multiple

The GCD of two numbers is the largest number that divides both. The Euclidean algorithm computes it in O(log(min(a,b))) using the property: gcd(a, b) = gcd(b, a % b).

Euclidean Algorithm
gcd(a, b):
  if b == 0: return a
  return gcd(b, a % b)

lcm(a, b) = (a * b) / gcd(a, b)   // avoid overflow: a / gcd(a,b) * b
Time: O(log min(a,b)) Space: O(log min(a,b)) stack
Interactive GCD Trace
Key Properties
  • gcd(a, 0) = a
  • gcd(a, b) = gcd(b, a % b)
  • lcm(a, b) × gcd(a, b) = a × b
  • Extended Euclidean: finds x, y such that ax + by = gcd(a,b)
  • Used in: fraction reduction, modular inverse, Bezout's identity

Sieve of Eratosthenes

Generate all primes up to N in O(N log log N)

Start with all numbers 2..N marked as prime. For each prime p, mark all multiples of p (starting from p²) as composite. Remaining unmarked numbers are prime.

Sieve of Eratosthenes
function sieve(n):
  is_prime = [true] * (n+1)
  is_prime[0] = is_prime[1] = false
  for p from 2 to sqrt(n):
    if is_prime[p]:
      for multiple from p*p to n step p:
        is_prime[multiple] = false
  return [i for i if is_prime[i]]
Time: O(N log log N) Space: O(N)

Prime Factorization

Decompose any number into prime factors

Trial division: divide by 2, then odd numbers up to √n. If n > 1 after the loop, n itself is prime. With a precomputed Smallest Prime Factor (SPF) array, factorization of any n ≤ N is O(log n).

Trial Division
function factorize(n):
  factors = []
  d = 2
  while d * d <= n:
    while n % d == 0:
      factors.append(d)
      n = n / d
    d++
  if n > 1: factors.append(n)
  return factors

// Example: 360 = 2³ × 3² × 5
Time: O(√n) With SPF sieve: O(log n)

Modular Arithmetic

Operations under a modulus — essential for large numbers in CP

In competitive programming, answers are often required modulo a prime (typically 10⁹+7 or 998244353). Modular arithmetic allows us to keep numbers small while computing correctly.

Key Properties
(a + b) % m = ((a % m) + (b % m)) % m
(a - b) % m = ((a % m) - (b % m) + m) % m  // +m to avoid negative
(a * b) % m = ((a % m) * (b % m)) % m
(a / b) % m = (a * modInverse(b, m)) % m   // only when gcd(b,m)=1

// Modular Inverse (Fermat's little theorem, m must be prime):
modInverse(a, m) = pow(a, m-2, m)

// Common mod: 1e9+7 = 1000000007 (prime)
Modular Inverse

The modular inverse of a under modulus m is b such that (a × b) ≡ 1 (mod m). Exists only when gcd(a, m) = 1. Computed via Fermat's little theorem when m is prime: a⁻¹ = aᵐ⁻² mod m.

Why 10⁹+7?
  • It is prime — modular inverse via Fermat's theorem works
  • Fits in a 32-bit integer (just under 2³¹)
  • Two values multiplied fit in a 64-bit integer (10⁹+7)² < 2⁶³

Fast Exponentiation

Binary exponentiation — compute aⁿ mod m in O(log n)

Naively computing aⁿ takes O(n) multiplications. Binary exponentiation uses the binary representation of n to compute it in O(log n) — squaring the base and multiplying when the current bit is 1.

Binary Exponentiation
function fastPow(base, exp, mod):
  result = 1
  base = base % mod
  while exp > 0:
    if exp % 2 == 1:          // current bit is 1
      result = result * base % mod
    exp = exp >> 1             // shift right (divide by 2)
    base = base * base % mod   // square the base
  return result

// Example: 2^10 = 1024
// Bits of 10: 1010
// Steps: base=2→4→16→256, multiply when bit=1
Time: O(log n) Space: O(1)

Euler's Totient Function

φ(n) — count of integers ≤ n coprime to n

φ(n) counts how many integers from 1 to n share no common factor with n (are coprime). This is foundational in RSA encryption and Fermat's little theorem.

Formula & Computation
// φ(n) using prime factorization:
// If n = p1^a1 × p2^a2 × ...
// φ(n) = n × ∏(1 - 1/pi) for each distinct prime pi

function totient(n):
  result = n
  p = 2
  while p * p <= n:
    if n % p == 0:
      while n % p == 0: n //= p
      result -= result // p   // result *= (1 - 1/p)
    p++
  if n > 1: result -= result // n
  return result

// Key facts:
// φ(prime p) = p - 1
// φ(1) = 1
// Fermat: a^φ(n) ≡ 1 (mod n) when gcd(a,n)=1
Single n: O(√n) All up to N: O(N log log N)

Chinese Remainder Theorem

Solve a system of modular congruences

Given a system of congruences with pairwise coprime moduli, CRT guarantees a unique solution modulo the product of all moduli. Essential for combining results computed under different moduli.

CRT — System of Congruences
// Find x such that:
// x ≡ r1 (mod m1)
// x ≡ r2 (mod m2)
// ...
// where all mi are pairwise coprime

// For two congruences:
// x ≡ a (mod m),  x ≡ b (mod n),  gcd(m,n)=1
// x = a + m * ((b - a) * modInverse(m, n) % n)
// Solution is unique mod (m * n)

// Extended Euclidean for general case:
function crt2(a, m, b, n):
  // find x = a + m*k such that a + m*k ≡ b (mod n)
  diff = (b - a % n + n) % n
  inv  = modInverse(m % n, n)
  k    = diff * inv % n
  x    = a + m * k
  return x, m * n          // solution, new modulus
Classic Example
x ≡ 2 (mod 3) — remainder 2 when divided by 3
x ≡ 3 (mod 5) — remainder 3 when divided by 5
x ≡ 2 (mod 7) — remainder 2 when divided by 7
↓
x = 23 (mod 105) ✅
