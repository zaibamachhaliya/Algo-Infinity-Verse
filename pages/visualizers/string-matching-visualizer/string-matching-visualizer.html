📖 How Each Algorithm Works

KMP

Knuth-Morris-Pratt

Builds a failure function (prefix table) for the pattern. When a mismatch occurs, instead of restarting from the beginning, KMP jumps back only as far as the failure function says — never re-examines text characters.

O(n+m) total No backtracking on text Failure fn = O(m)
Z

Z-Algorithm

Constructs a Z-array where Z[i] = length of the longest substring starting at position i that matches a prefix of the string. Run on pattern + '$' + text — any Z[i] ≥ m is a match.

O(n+m) total Single pass Z-box window
RK

Rabin-Karp

Computes a rolling hash of each text window. Only does full character comparison when hashes match — making it O(n) average. Worst case O(nm) if all hashes collide (spurious hits).

O(n) average Rolling hash Good for multi-pattern
