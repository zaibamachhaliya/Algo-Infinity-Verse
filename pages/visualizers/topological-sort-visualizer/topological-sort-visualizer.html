Topological Ordering & DAG Concepts

What is a Directed Acyclic Graph (DAG)?

A **Directed Acyclic Graph (DAG)** is a graph with directed edges and no cycles. This means there is no path starting at any node $V$ that follows the direction of the edges to loop back to $V$.

A **Topological Sort** is a linear ordering of vertices such that for every directed edge $U \rightarrow V$, vertex $U$ comes before $V$ in the ordering. Topological sort is only possible if and only if the graph is a **DAG**. If a cycle exists, no valid topological ordering exists.

Algorithms comparison

**Kahn's Algorithm (BFS)** works by continually identifying nodes with an **indegree of 0** (no incoming dependencies), appending them to a queue, and "removing" their outgoing connections. If we finish sorting and some nodes have not been visited, a cycle must exist.

**DFS Topological Sort** runs a Depth First Search from each unvisited node. As we backtrack (finish visiting all descendants of node $V$), we push $V$ onto a Stack. If we encounter a node currently being visited (in the recursion stack), we detect a back edge, verifying the graph has a cycle.
