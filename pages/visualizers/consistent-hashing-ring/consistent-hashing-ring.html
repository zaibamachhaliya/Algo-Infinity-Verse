The Teachable Moment

🔄 Why Virtual Nodes Matter

🎯

Without Virtual Nodes

With 1 position per physical server, ring gaps are essentially random. One server might own 5% of keyspace, another 40%. Adding a new server helps only its immediate clockwise neighbor — everyone else's load is untouched, so imbalance can persist indefinitely.

📊

With Virtual Nodes

Each physical server claims V random positions. With V=20+, the law of large numbers smooths ownership to within a few percent of 1/N per server — regardless of how the random hash positions happen to land.

🔀

The Migration Guarantee

Consistent hashing's core promise: adding or removing 1 of N servers moves only ~K/N keys (K = total keys), regardless of virtual node count. Virtual nodes fix the LOAD BALANCE problem without breaking this migration guarantee — moved keys still come only from immediate ring neighbors.

🏗️

Real Systems

DynamoDB uses 100-200+ virtual nodes per physical partition. Cassandra historically used 256 tokens per node by default. Chord DHT assigns each node's virtual identifiers via consistent hashing of its IP+port, forming the basis of BitTorrent's distributed tracker.
