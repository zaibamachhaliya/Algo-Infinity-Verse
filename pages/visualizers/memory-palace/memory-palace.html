🧠 Why Memory Layout Matters

⚡

Cache Locality

Arrays are contiguous — each access likely hits the CPU cache. Linked lists scatter across RAM — every pointer dereference is a potential cache miss. This is why array iteration can be 100× faster than linked list traversal even at the same O(n) complexity.

🔀

Memory Fragmentation

After many alloc/free cycles, RAM develops gaps too small for new allocations. Contiguous structures (arrays) suffer from this — they need a single large block. Linked structures tolerate fragmentation since each node can live anywhere.

👉

Pointer Overhead

Every linked list node stores extra pointer(s) — 8 bytes each on 64-bit systems. A doubly-linked list of 1M integers uses ~24MB instead of 4MB for an array. Hash maps store both key pointers and value pointers in each bucket.

🌳

Heap-Indexed Trees

Binary trees stored as arrays (heap indexing) eliminate pointer overhead entirely. Parent of i = ⌊(i-1)/2⌋. Left child = 2i+1. Right child = 2i+2. This is why binary heaps are so cache-efficient despite being "trees".
