🦠 Gossip Protocols in the Wild

🗣️

Apache Cassandra

Cassandra uses gossip every second — each node contacts up to 3 random peers to exchange cluster state (who's alive, who's dead, what ring positions exist). No coordinator needed. A new node joining the cluster propagates its existence to all nodes within O(log n) gossip rounds.

🧮

The SIR Model

Susceptible → Infected → Recovered. Each round, an infected node contacts K random neighbors (fanout). If a susceptible neighbor receives the message, it becomes infected. After spreading its data, a node becomes "recovered" (still functional, just done spreading). The math governing convergence speed is identical to epidemic spread equations.

🔄

Anti-Entropy Repair

After convergence, Cassandra still runs gossip — not for new data, but for "anti-entropy repair." Nodes periodically exchange Merkle tree digests to find which data is inconsistent and sync it. This is why Cassandra can heal from network partitions automatically, without any operator intervention.

📡

Push vs Pull vs Push-Pull

Push: sender actively contacts neighbors (fast early spread). Pull: susceptible nodes ask peers (slower early spread, better late convergence). Push-Pull: both simultaneously — each exchange is bidirectional. Amazon Dynamo and CockroachDB use Push-Pull because it converges in half as many rounds as pure Push for the last few susceptible nodes.
