📖 How Skip Lists Work

01

The Idea

A skip list is multiple linked lists stacked on top of each other. The bottom level has all nodes. Higher levels are "express lanes" containing only some nodes — allowing search to skip large sections.

O(log n) averageProbabilistic
02

Level Assignment

When inserting, each new node flips a coin. Heads → the node also appears in the next level up. This continues until tails. Result: each level has ~half the nodes of the level below.

P(level k) = p^kExpected O(log n) levels
03

Search — O(log n)

Start at the topmost level, leftmost node. Move right until the next value exceeds target. Drop down one level. Repeat. The express lanes skip most nodes — like a binary search on a linked list.

Top-downRight then down
04

Used In Production

Redis uses skip lists for its sorted sets (ZADD/ZRANGEBYSCORE). LevelDB and RocksDB use skip lists for in-memory memtables. Apache Lucene uses them for index structures.
