Min-Cost Max-Flow Concepts

Successive Shortest Path Algorithm

The **Successive Shortest Path** algorithm is a classic method to solve the Min-Cost Max-Flow problem. It repeatedly finds the shortest path in the **residual graph** from source $S$ to sink $T$ and augments flow along it.

In this context, the "length" or "distance" of an edge is its **unit cost** ($u$). Since residual back-edges have negative costs (to allow canceling flow), we must use a shortest path algorithm that can handle negative edge weights, such as **Bellman-Ford** or **SPFA**.

Residual Network & Cost Canceling

For each directed edge with flow $f$, capacity $c$, and unit cost $u$, the residual graph contains: - A **forward edge** with capacity $c - f$ and cost $u$. - A **backward edge** with capacity $f$ and cost $-u$.

When flow is pushed along a backward edge, it effectively reduces the flow on the original edge. Because the backward edge has a cost of $-u$, this cancellation actually **reduces** the total cost, showing how the algorithm balances cost and flow.
