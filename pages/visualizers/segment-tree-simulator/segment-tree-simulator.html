📖 How Segment Trees Work

01

Build — O(n)

Each leaf node stores one array element. Internal nodes store the aggregate (sum/min/max) of their children's ranges. The tree has 2n−1 nodes total.

Time: O(n) Space: O(n)
02

Range Query — O(log n)

At each node, check: fully inside range → return value. No overlap → return 0. Partial → recurse into both children. At most 4 nodes per level are visited.

Time: O(log n) Space: O(log n)
03

Point Update — O(log n)

Find the leaf, update it, then propagate upward: each parent recalculates its value from children. Exactly one path from leaf to root is touched.

Time: O(log n) Space: O(1)
04

vs Prefix Sum Array

Prefix Sum: O(1) query but O(n) update. Segment Tree: O(log n) both. Use Segment Tree when you need both queries and updates.

Both O(log n)

🌍 Real-World Applications

📊 Range Sum Queries

Database aggregations — sum sales between date A and date B, with live updates as new sales come in.

🎮 Game Development

Collision detection over intervals, health bar range queries, and live entity updates on large game maps.

🏆 Competitive Programming

The go-to structure for interval problems — range min, range max, range sum, range GCD, lazy propagation.

📈 Stock Analytics

Range maximum/minimum price over a date window, with O(log n) updates as each new price tick arrives.
