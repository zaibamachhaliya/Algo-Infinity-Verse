📖 How It Works

01

Online Construction

Unlike suffix trees (often built via complex offline algorithms like Ukkonen's), the suffix automaton extends naturally one character at a time. Each new character either extends existing states or triggers a clone when a transition's length doesn't match exactly.

02

The Clone State

When following suffix links during extension, if we hit a state whose length doesn't fit (len[p]+1 ≠ len[q]), we clone that state — splitting it into a shorter version. This is the single trickiest part of the algorithm, and this tool animates it explicitly.

03

Counting All Substrings

Every state represents a range of substring lengths [len[link]+1, len[state]] that all end at the same set of positions. Summing (len[state] − len[link[state]]) over all states gives the exact count of distinct substrings — in O(n) states, not O(n²) substrings.

04

Linear-Time LCS

Build the automaton once on string A. Stream string B through it: follow transitions when possible, fall back via suffix links on mismatch. Track the longest match length at any point — that's the LCS, found in O(|A|+|B|) total.
