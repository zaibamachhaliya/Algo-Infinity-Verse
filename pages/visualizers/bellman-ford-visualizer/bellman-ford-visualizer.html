📖 How They Work

BF

Bellman-Ford — O(VE)

Relaxes every edge V−1 times. After pass k, all shortest paths using at most k edges are correct. On pass V, if any distance still decreases — a negative cycle exists.

Time: O(VE) Space: O(V) Negative edges: ✅ Negative cycle detect: ✅
SPFA

SPFA — O(kE) average

Queue-based Bellman-Ford. Only re-relaxes neighbors of nodes whose distance improved. Average O(E), worst case O(VE). Detects negative cycles via visit count per node.

Avg: O(kE) Worst: O(VE) Queue-based Faster in practice
VS

vs Dijkstra

Dijkstra is O((V+E) log V) but fails on negative edges. Bellman-Ford handles negatives but is slower. Use BF when graph has negative weights; Dijkstra otherwise.

Dijkstra: no negatives BF: handles negatives BF detects neg cycles
