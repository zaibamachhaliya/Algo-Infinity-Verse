1. The Intuition: Why HLD?

When dealing with flat arrays, answering range queries (like max, min, or sum) is trivial with a Segment Tree in $O(\log N)$. However, a Tree is not flat. If you need to find the maximum edge weight on a path between node $U$ and node $V$, a standard DFS takes $O(N)$ per query. For $10^5$ queries, this will instantly result in a Time Limit Exceeded (TLE) error.

Heavy-Light Decomposition (HLD) solves this by slicing the tree into a set of continuous, non-overlapping linear paths (chains). Once flattened, we map these chains onto a 1D array and unleash a standard Segment Tree on top of it. This drops our query time to a blistering $O(\log^2 N)$.

2. Heavy vs Light Edges

How do we decide which paths form the "chains"? We do it based on subtree size to guarantee mathematical efficiency.

  • Heavy Edge: Connects a node to the child that has the largest subtree size. (Max 1 per node).
  • Light Edge: Connects a node to any of its other children.
The Golden Theorem: Any path from the root to a leaf node will contain at most $O(\log N)$ light edges. Because every time you traverse a light edge, the subtree size at least halves: $size(v) \le size(u) / 2$.

4. DFS Construction (Two-Pass System)

We construct the decomposition using two Depth-First Searches.

Pass 1: Subtree Sizing

We calculate depths, parents, and sizes to identify the heavy child for every node.

void dfs1(int v = 1, int p = 0, int d = 0) {
    parent[v] = p;
    depth[v] = d;
    sz[v] = 1;
    for (int &to : adj[v]) {
        if (to == p) continue;
        dfs1(to, v, d + 1);
        sz[v] += sz[to];
        // Swap to keep the heavy child at the front of the adjacency list
        if (sz[to] > sz[adj[v][0]]) swap(to, adj[v][0]);
    }
}

Pass 2: Flattening the Tree

We traverse the tree again, prioritizing the heavy children to ensure they occupy contiguous indices in our 1D array (`pos` array).

void dfs2(int v = 1, int p = 0, int curr_head = 1) {
    head[v] = curr_head;  // The head of the chain this node belongs to
    pos[v] = timer++;     // The node's position in the 1D Segment Tree array
    
    for (int to : adj[v]) {
        if (to == p) continue;
        // If it's the heavy child, it continues the current chain
        if (to == adj[v][0]) dfs2(to, v, curr_head);
        // If it's a light child, it starts a brand new chain
        else dfs2(to, v, to);
    }
}

5. Routing Queries $O(\log^2 N)$

To query the path between $A$ and $B$, we jump up the tree along the heavy chains until both nodes are on the same chain.

int query_path(int a, int b) {
    int res = 0;
    while (head[a] != head[b]) {
        if (depth[head[a]] < depth[head[b]]) swap(a, b);
        // Query the segment tree for the current chain
        res = max(res, segTree.query(pos[head[a]], pos[a]));
        a = parent[head[a]]; // Jump the light edge
    }
    if (depth[a] > depth[b]) swap(a, b);
    res = max(res, segTree.query(pos[a], pos[b]));
    return res;
}
