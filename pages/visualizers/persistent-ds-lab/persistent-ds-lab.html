📖 How Path Copying Works

01

Path Copying

To insert a value into a persistent BST, copy only the nodes on the path from the root to the insertion point — O(log n) nodes. All other subtrees are shared with the previous version by reference. The new root becomes the root of version n+1.

02

Structural Sharing

After k insertions into a tree of height h, the total nodes created is k × h, not k × n. In a balanced BST with 1000 elements, 10 insertions creates only ~100 new nodes, not 10,000. The other 9,900 references point to existing nodes.

03

Immutability

Because no node is ever modified after creation, any code holding a reference to version n sees a stable, unchanging tree forever. This makes persistent structures inherently thread-safe and enables trivial undo — just keep a list of old roots.

04

Real Uses

Git stores every commit as an immutable tree of blobs and subtrees, sharing unchanged directories. Clojure's vectors and maps are persistent. React's virtual DOM reuses unchanged subtrees. Haskell's data structures are persistent by default.
