🔬 What Each Stage Does

01

Tokenization (Lexing)

The lexer reads your source character by character and groups them into tokens — the atoms of the language. for becomes a KEYWORD token. 42 becomes a NUMBER token. + becomes an OPERATOR token. No structure yet — just atoms.

02

Parsing (AST Construction)

The parser takes the token stream and applies grammar rules to build a tree. A for keyword followed by ( and a semicolon-separated expression becomes a ForStatement node with Init, Test, and Update children.

03

Optimization

The optimizer walks the AST looking for improvements: constant folding replaces 2 + 3 with 5, dead code elimination removes unreachable if (false) branches, loop invariant hoisting moves expressions that don't change inside loops to before the loop.

04

Bytecode Generation

The optimized AST is walked again to emit bytecode — a sequence of simple instructions (LOAD, STORE, ADD, JUMP, CALL, RETURN) that a virtual machine like V8 or the JVM executes directly. Bytecode is portable and faster to execute than re-parsing source.
