01
Stage 1

Understand the Problem

Before writing a single line of code, you must fully understand what the problem is asking. Most mistakes happen because candidates rush this step.

The 4-Step Clarification Process

  1. Read the problem carefully — Read it 2-3 times. Circle key terms.
  2. Identify inputs and outputs — What are the function parameters? What should it return?
  3. Determine constraints — Input size, value ranges, time/space limits.
  4. Ask clarifying questions — Can the input be empty? Are there duplicates? Is it sorted?
💡

Pro Tip: Restate the problem in your own words and confirm with the interviewer. "So just to confirm, I need to find the first non-repeating character in a string and return its index — is that correct?"

Example: Clarifying Questions

Given: Given an array of integers, return indices of two numbers that add up to a target.

  • What if no two numbers sum to the target?
  • Can the same element be used twice?
  • Is the array sorted?
  • What if there are multiple valid pairs?
Practice: Ask Questions

For the problem "Find the longest substring without repeating characters", what clarifying questions would you ask?

  • What characters are allowed? (ASCII, Unicode, only letters?)
  • What should be returned for an empty string?
  • Should substring be contiguous? (Yes, by definition)
  • Case-sensitive or case-insensitive?
  • What is the maximum length of the input string?
02
Stage 2

Brute Force Solution

Start with the simplest possible solution. It doesn't need to be efficient — it needs to be correct. A brute force solution shows you understand the problem and gives you a baseline to optimize from.

Why Start With Brute Force?

  • Guarantees you can solve the problem (even if slowly)
  • Provides a correctness baseline for optimized solutions
  • Demonstrates your thought process to the interviewer
  • Often suggests where the optimization opportunities lie
⚡

Always state the complexity — After writing brute force, analyze it: "This is O(n²) time and O(1) space. Can we do better?"

Example: Two Sum — Brute Force

Problem: Given an array nums and a target, return indices of two numbers that add up to the target.

Approach: Compare every pair of elements.

JavaScript 
function twoSum(nums, target) {
  for (let i = 0; i < nums.length; i++) {
    for (let j = i + 1; j < nums.length; j++) {
      if (nums[i] + nums[j] === target) {
        return [i, j];
      }
    }
  }
  return [];
}
Complexity Analysis
Time Complexity: O(n²) — nested loops over the array
Space Complexity: O(1) — only using a few variables
03
Stage 3

Optimize Your Solution

Once you have a working brute force, identify the bottlenecks and apply the right patterns to improve time and space complexity.

How to Optimize

  1. Identify the bottleneck — usually the innermost loop
  2. Look for redundant work — are you recomputing values?
  3. Consider a better data structure — HashMap, Set, Heap, etc.
  4. Apply a known pattern — Sliding Window, Two Pointers, etc.
  5. Evaluate the time-space tradeoff

Optimization 1: HashMap Lookup

Replace the inner loop with a HashMap to track seen numbers. Trade O(n) space for O(1) lookup.

JavaScript — Hash Map 
function twoSum(nums, target) {
  const map = new Map();
  for (let i = 0; i < nums.length; i++) {
    const complement = target - nums[i];
    if (map.has(complement)) {
      return [map.get(complement), i];
    }
    map.set(nums[i], i);
  }
  return [];
}
Complexity Analysis
Time Complexity: O(n) — single pass through the array
Space Complexity: O(n) — HashMap stores up to n elements

Common Optimization Patterns

HashMap
O(1) lookup trades space for speed
Sliding Window
O(n) for contiguous subarray problems
Prefix Sum
O(1) range sum queries
Binary Search
O(log n) on sorted data
04
Stage 4

Write Clean Code

With a clear optimized plan, now write code that is readable, maintainable, and correct. Your solution should communicate intent as clearly as it computes results.

Clean Coding Principles

  • Use descriptive names — numberOfOccurrences not n, numMap not mp
  • Write modular logic — Extract helper functions for reusable logic
  • Handle edge cases first — Check null, empty, invalid inputs at the top
  • Keep functions small — Each function should do one thing well
  • Add comments sparingly — Prefer self-documenting code
JavaScript — Clean Version 
function twoSum(nums, target) {
  const seen = new Map();

  for (let i = 0; i < nums.length; i++) {
    const complement = target - nums[i];

    if (seen.has(complement)) {
      return [seen.get(complement), i];
    }

    seen.set(nums[i], i);
  }

  return [];
}
⚠️

Edge Cases Checklist: Empty input • Single element • Null/undefined • Large values • Negative numbers • Duplicates

Practice: Edge Cases

What edge cases should the Two Sum function handle?

  • Empty array: Return [] (no pair possible)
  • Single element: Return [] (need at least 2 elements)
  • No valid pair: Return []
  • Negative numbers: The complement logic works naturally with negatives
  • Target is zero: e.g., [1, -1], target 0 → [0, 1]
  • Multiple valid pairs: Return the first one found (any is valid per spec)
05
Stage 5

Test Your Solution

Testing is not optional. Walk through your code manually with example inputs and verify correctness. A methodical testing strategy catches bugs before the interviewer does.

Testing Strategy

  1. Test the example given in the problem — verify your understanding
  2. Test edge cases — empty, single, boundary values
  3. Test duplicate values — ensure correct handling
  4. Test large inputs — reason about time/space constraints
  5. Trace through the code — simulate execution with small inputs

Problem 1 Easy

Contains Duplicate

Given an integer array nums, return true if any value appears at least twice, otherwise return false.

Example: nums = [1, 2, 3, 1] → true
Problem 2 Medium

Maximum Subarray

Given an integer array nums, find the contiguous subarray (at least one element) with the largest sum and return that sum.

Example: nums = [-2, 1, -3, 4, -1, 2, 1, -5, 4] → 6
Problem 3 Hard

Sliding Window Maximum

Given an array nums and an integer k, return the maximum element in each sliding window of size k.

Example: nums = [1, 3, -1, -3, 5, 3, 6, 7], k = 3 → [3, 3, 5, 5, 6, 7]
